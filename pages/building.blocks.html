Cascade and Inheritance

Cascade:

If two rules are equal, the one that comes last is applied.

Specificity:

The rule is applied if it is the most specific to that element.

Inheritance:

Some properties set on parent elements are inherited by their child elements, and some aren't. For example, color is inherited, width is not.

inherit: Turns inheritance on.

initial: Turns inheritance off.

unset: Resets to it's natural value, whether that is inherit or initial.


Selectors

Type, class and ID:

This group includes selectors that target an HTML element.
Examples: h1{}, .a-class{}, #some-id{}.

Attribute:

This group of selectors selects elements based on the presence of a certain attribute.
Examples: a[title]{}, a[href="example.com"]{}.

Pseudo-class and pseudo-element:

This group of selectors styles certain states or parts of an element.
Examples: a:hover{}, p::first-line{}.

Combinator:

This group of selectors combine other selectors in order to target elements within our documents.
Example: article > p{}.

Universal:

This selects all elements: *{}

Descendant:

The descendant combinator combines two selectors such that elements matched by the second selector are selected if they have an ancestor element matching the first selector.
Examples: div li{}, p strong{}.

Adjacent sibling:

This combinator matches the second element only if it immediately follows the first element. Example: img + p{}.

General sibling:

Same as the adjacent sibling combinator, but without the "immediately" part.
Example: img ~ p{}.

The box model

Block boxes:

  • Will break onto a new line;
  • Will fill all of the space available in that line;
  • The width and height properties are respected;
  • Padding, margin and border will push other elements away from the box.

Inline boxes:

  • Will not break onto a new line;
  • The width and height properties are not respected;
  • Vertical padding, margins and borders will apply but will not cause other inline boxes to move away from the box.
  • Horizontal padding, margins and borders will apply and will cause other inline boxes to move away from the box.

Content box:

The area where the content is displayed. Can be changed using width and height.

Padding box:

The area that sits around the content. Can be changed using padding.

Border box:

The area that wraps content and padding. Can be changed using border.

Margin box

The area that wraps everything as whitespace between this element and other elements. Can be changed using margin.

Standard CSS box model:

The size of the box is the size of the content box.

Alternative CSS box model:

The size of the box is the size of the content, padding and margin boxes together.
Set box-sizing: border-box to achieve this.

Margins:

If two margins collapse, the highest absolute value margin will be the one that's displayed. However, if one of them is positive and the other is negative, the displayed value will be the sum of the two.

Borders:

If the box sizing is standard, the border is added to the width and height of the box. If the box sizing is border-box, the border takes up the available width and height.
The border can be styled using border-width, border-style, border-color and border-radius to round off corners.

Padding:

Unlike margins, padding cannot be negative. If the box has a background, it will be displayed on the padding.

Display: inline-block

Using this display setting will give the box these properties:

  • The box will not break into another line;
  • The width and height properties are respected;
  • Vertical padding, borders and margins will push other elements away.

Backgrounds and borders

Background color:

The background-color attribute gives the element's background a color.

Background image:

The background-image attribute displays and image on the element's background.
The background-repeat attribute controls the tiling behaviour of the image:

  • background-repeat: no-repeat -- the background doesn't repeat;
  • background-repeat: repeat-x -- repeats horizontally;
  • background-repeat: repeat-y -- repeats vertically;
  • background-repeat: repeat -- the default, repeats in both directions.

The background-size attribute controls the sizing of the image:

  • background-size: cover -- completely covers the box (may end up cropped).
  • background-size: contain -- fits inside the box (may cause gaps).
  • You can also use pixels or percentages.

The background-position attribute controls the position of the image. Use values like top, center or specify values to be applied to x and y, like 25% 75%.

You can also use gradients for your background. Just use the background-image attribute and assign a gradient value to it, such as linear-gradient(#f69d3c, #3f87a6).

The background-attachment attribute controls the image's behaviour when scrolling:

  • background-attachment: scroll -- scrolls with the page and the element.
  • background-attachment: fixed -- does not scroll.
  • background-attachment: local -- scrolls with the page, but not the element.
