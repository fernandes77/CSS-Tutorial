Wrapping

Your flexbox items content might overflow the parent's width or height. The flex-wrap attribute, when set to wrap, breaks the items into a new line whenever there is content overflow.
You can also set flex: 200px to the <article> elements to specify a minimum width.

Horizontal and vertical alignment

The align-items and justify-content attributes align flex items.
align-items controls the cross axis while justify-content controls the main axis.

{display: flex;}
  • align-items: stretch -- default value, occupies the container's height;
  • align-items: center -- placed in the center vertically, occupies the content's height;
  • align-items: flex-start/flex-end -- puts the items up or down, respectively.
  • justify-content: flex-start -- default value, makes items sit at the start of the main axis;
  • justify-content: flex-end -- makes items sit at the end of the main axis;
  • justify-content: center -- makes items sit at the center of the main axis;
  • justify-content: space-around -- will distribute all items evenly across the main axis, with a bit of space at the start and at the end;
  • justify-content: space-between -- will distribute all items evenly across the main axis, without space at the start or at the end.
{display: flex; align-items: center; justify-content: space-around;}

Ordering flex items

The order attribute sets the order in which flex items are displayed. The default value is 0.
The values can be negative too. If two items have the same value, they will follow the source code order.

.order:first-child {order: 1;}

The implicit and explicit grid

When you set grid-template-columns or grid-template-rows, you are creating an explicit grid. When content is placed outside of that grid, the implicit grid is created.
The implicit grid will be as large as the content in it. To determine a size for it, define grid-auto-rows or grid-auto-columns depending on which is implicit.

Responsive design

As more diverse screen sizes became available, the concept of responsive web design (RWD) appeared, a set of practices that allows web pages to alter their layout and appearance to suit different screen widths, resolutions, etc.

Media queries

Media queries allow us to run tests and alter our layout depending on the characteristics of the device it is being rendered on. 

      
        @media screen and (min-width: 800px) { 
          .container { 
            margin: 1em 2em; 
          } 
        }

Flexible grids

Using flexible grids, you only need to alter your layout when it starts to look bad. It is a legacy layout method.

Modern layout technologies

  • Multicol: If column-count is used, the column width will change accordingly. And vice-versa.
  • Flexbox: Flexbox items will shrink and grow according to the space in their container. Change the proportions of each item using flex.
  • CSS grid: The fr unit allows for distribution of available space across grid tracks.

Responsive images

The <picture> tag allows you to change how the image is displayed depending on the size of the screen.

Responsive typography

A good approach is to use the viewport unit vw to enable responsive typography. Use it in conjunction with rem with the calc() function, like this: 

      
        h1 {
          font-size: calc(1.5rem + 3vw);
        }

The viewport meta tag

Always use the following meta tag: 

      
        <meta name="viewport" content="width=device-width,initial-scale=1">

This meta tag tells mobile browsers that they should set the width of the viewport to the device width, and scale the document to 100% of its intended size, which shows the document at the mobile-optimized size that you intended.
